## 3. Безопасность и секреты

### Что проверено

- Поиск паттернов API-ключей (`sk-…`, `ghp_…`) в отслеживаемых файлах — **не найдено**.
- `.env` в `.gitignore` — **да**.
- `.env.example` содержит только плейсхолдеры — **да**.
- Локальный `.env` существует, но **не** в индексе Git.

### Важное предупреждение (история Git)

В истории репозитория файл `.env` **ранее коммитился** и был удалён коммитом:

`f48e51f — Remove .env from git tracking - secrets should not be committed`

**Рекомендация:** если в том коммите были реальные ключи, их необходимо **отозвать/ротировать** в консоли Proxy API / OpenAI. Удаление файла из текущей ветки не стирает секреты из истории Git без `git filter-repo` / BFG.

### Внесённые меры

- Усилен `.gitignore`: `.env.*`, `*.pem`, `*.key`, `credentials.json`.
- Убран fallback `"dummy-key"` — при отсутствии ключа приложение завершается с явной ошибкой.
- Валидация `PROXY_API_KEY` при `API_PROVIDER=proxy`.
- Валидация допустимых значений `API_PROVIDER` (`proxy` | `openai`).
